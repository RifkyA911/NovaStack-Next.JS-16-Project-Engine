# Dashboard / Admin Panel

Comprehensive dashboard and admin panel system with multiple modules for managing your application.

## Overview

The dashboard system provides a full-featured admin interface with modules for inbox management, inventory tracking, order processing, user management, reporting, and settings. Built with modern React components and integrated with the DataTable component for data display.

## Configuration

### RBAC Configuration

Dashboard access is controlled by RBAC middleware:

```typescript
// src/middlewares/role.ts
const userForbiddenRoutes: Record<string, string[]> = {
  administrator: ["!"], // Full access
  user: ["/dashboard/users"], // Restricted from user management
  guest: ["/dashboard"], // No dashboard access
};
```

### Navigation Configuration

Sidebar navigation is configured in the dashboard sidebar component:

```typescript
// src/app/dashboard/_components/dashboard-sidebar.tsx
const navigation = [
  { name: 'Overview', href: '/dashboard', icon: HomeIcon },
  { name: 'Inbox', href: '/dashboard/inbox', icon: InboxIcon },
  { name: 'Inventory', href: '/dashboard/inventory', icon: PackageIcon },
  { name: 'Orders', href: '/dashboard/orders', icon: ShoppingCartIcon },
  { name: 'Users', href: '/dashboard/users', icon: UsersIcon },
  { name: 'Reports', href: '/dashboard/reports', icon: ChartIcon },
  { name: 'Settings', href: '/dashboard/settings', icon: SettingsIcon },
];
```

## Best Practices

1. **Use DataTable**: Always use the DataTable component for data display
2. **Consistent Layout**: Follow the established layout pattern
3. **Error Handling**: Implement proper error handling for API calls
4. **Loading States**: Show loading indicators during data fetch
5. **Responsive Design**: Ensure all modules work on mobile devices
6. **Accessibility**: Follow WCAG guidelines for accessibility
7. **Performance**: Implement pagination for large datasets
8. **Security**: Always validate permissions on server-side

## Customization

### Adding New Modules

1. Create new module directory in `src/app/dashboard/`
2. Create page.tsx with module content
3. Add navigation item to dashboard sidebar
4. Configure RBAC permissions if needed
5. Add to documentation

### Customizing Dashboard Components

1. Modify component files in `src/app/dashboard/_components/`
2. Follow existing component patterns
3. Maintain responsive design
4. Test across different screen sizes

## Troubleshooting

### Dashboard not loading
- Check RBAC permissions
- Verify authentication status
- Check browser console for errors
- Ensure all dependencies are installed

### DataTable not displaying data
- Verify data structure matches column definitions
- Check API endpoints
- Ensure proper error handling
- Verify pagination configuration

### Navigation not working
- Check sidebar navigation configuration
- Verify route paths
- Check for JavaScript errors
- Ensure proper routing setup
