# Authentication Configuration

Complete authentication system using NextAuth.js with multiple providers and custom pages.

## Overview

The authentication system provides a secure, flexible authentication solution using NextAuth.js. It supports multiple authentication providers, custom pages, and seamless integration with the RBAC middleware for role-based access control.

## Configuration

### NextAuth Configuration

```typescript
// app/api/auth/[...nextauth]/route.ts
export const { handlers, auth, signIn, signOut } = NextAuth({
  providers: [
    GoogleProvider({
      clientId: process.env.GOOGLE_CLIENT_ID,
      clientSecret: process.env.GOOGLE_CLIENT_SECRET,
    }),
    CredentialsProvider({
      // Credentials configuration
    }),
  ],
  pages: {
    signIn: "/auth/login",
    error: "/auth/error",
    signOut: "/auth/signout",
    verifyRequest: "/auth/verify-request",
    newUser: "/auth/new-user",
  },
  callbacks: {
    async jwt({ token, user }) {
      if (user) {
        token.role = user.role;
      }
      return token;
    },
    async session({ session, token }) {
      session.user.role = token.role;
      return session;
    },
  },
});
```

## Authentication Flow

### Login Flow

1. User navigates to `/auth/login`
2. User enters credentials or clicks Google OAuth
3. Form validation and reCAPTCHA verification
4. NextAuth authenticates user
5. JWT token is created with user role
6. User is redirected to dashboard or intended page
7. RBAC middleware checks permissions

### Registration Flow

1. User navigates to `/auth/register`
2. User fills registration form
3. Form validation and reCAPTCHA verification
4. User account is created
5. Email verification sent (if enabled)
6. User is redirected to login or verification page
7. User can then login with credentials

### Password Reset Flow

1. User navigates to `/auth/forgot-password`
2. User enters email address
3. System generates reset token
4. Reset link sent to email
5. User clicks link in email
6. User enters new password
7. Password is updated
8. User can login with new password

### Access Page Middleware

```typescript
// src/middlewares/auth.ts
export async function accessPageMiddleware(req: NextRequest) {
  const token = await getToken({
    req,
    secret: process.env.NEXTAUTH_SECRET,
  });

  // Redirect authenticated users away from auth pages
  if (token && req.nextUrl.pathname.startsWith("/auth")) {
    return NextResponse.redirect(new URL("/dashboard", req.url));
  }

  return NextResponse.next();
}
```

## RBAC Integration

### Role Assignment

User roles are assigned during registration and stored in the JWT token:

```typescript
// In your registration handler
const user = await createUser({
  email,
  password,
  role: 'user', // or 'administrator', 'moderator', etc.
});

// Role is added to JWT token in NextAuth callback
callbacks: {
  async jwt({ token, user }) {
    if (user) {
      token.role = user.role;
    }
    return token;
  },
}
```

### Permission Checking

```typescript
// In your components or API routes
import { auth } from "@/auth/config";

const session = await auth();
const userRole = session?.user?.role;

if (userRole !== 'administrator') {
  // Deny access
}
```

## Best Practices

1. **Environment Variables**: Never commit secrets to version control
2. **Strong Passwords**: Enforce strong password requirements
3. **HTTPS**: Always use HTTPS in production
4. **Rate Limiting**: Implement rate limiting on auth endpoints
5. **Session Management**: Implement proper session timeout
6. **Error Handling**: Provide user-friendly error messages
7. **Logging**: Log authentication attempts for security monitoring
8. **Testing**: Thoroughly test all authentication flows

## Troubleshooting

### Login not working
- Check NEXTAUTH_SECRET is set correctly
- Verify NextAuth configuration
- Check provider credentials (Google OAuth)
- Ensure database connection for user storage
- Check browser console for errors

### OAuth not working
- Verify OAuth client ID and secret
- Check redirect URIs in OAuth provider console
- Ensure NEXTAUTH_URL matches your domain
- Check CORS configuration
- Verify OAuth provider status

### Session not persisting
- Check cookie configuration
- Verify NEXTAUTH_SECRET consistency
- Check browser cookie settings
- Ensure proper session callback implementation
- Check token expiration settings

### reCAPTCHA not working
- Verify site key and secret key
- Check domain configuration in reCAPTCHA console
- Ensure proper script loading
- Check for JavaScript errors
- Test with different browsers
