# DialogForm Component

Dynamic dialog form component built on top of [TanStack Form](https://tanstack.com/form/latest) and [Zod](https://zod.dev/) validation. Provides a flexible, reusable form-in-dialog solution with built-in validation and file upload support.

## Features

### Core Functionality
- **Dynamic Fields** - Configure form fields through simple configuration objects
- **Validation** - Built-in Zod schema validation with real-time error messages
- **Field Types** - Support for text, email, tel, textarea, and file inputs
- **File Upload** - Integrated file upload component with drag-and-drop support
- **Controlled/Uncontrolled** - Support for both controlled and uncontrolled dialog states
- **Loading States** - Built-in loading indicator during form submission
- **Form Reset** - Automatic form reset when dialog closes

### Visual Enhancements
- **Error States** - Visual feedback for validation errors
- **Loading Indicator** - Spinner during form submission
- **Responsive Design** - Mobile-friendly dialog layout
- **Custom Styling** - Full control over appearance through props

### Advanced Features
- **TanStack Form Integration** - Modern form state management
- **Zod Validation** - Type-safe validation with excellent DX
- **File Upload Integration** - Seamless file handling with preview
- **Accessibility** - WCAG compliant with proper ARIA labels
- **TypeScript** - Full type safety with generics

## Quick Start

```tsx
import { DynamicDialogForm } from "@/components/organisms/dialog-form/dialog-form";
import { z } from "zod";

// Define validation schema
const userSchema = z.object({
  name: z.string().min(2, "Name must be at least 2 characters"),
  email: z.string().email("Invalid email address"),
  message: z.string().min(10, "Message must be at least 10 characters"),
});

// Define form fields
const fields = [
  {
    name: "name",
    label: "Full Name",
    type: "text" as const,
    placeholder: "Enter your name",
    validation: userSchema.shape.name,
  },
  {
    name: "email",
    label: "Email Address",
    type: "email" as const,
    placeholder: "you@example.com",
    validation: userSchema.shape.email,
  },
  {
    name: "message",
    label: "Message",
    type: "textarea" as const,
    placeholder: "Write your message...",
    validation: userSchema.shape.message,
  },
];

// Handle form submission
const handleSubmit = async (data: any) => {
  console.log("Form data:", data);
  // Send to API
};

// Use the component
<DynamicDialogForm
  title="Contact Form"
  description="Send us a message and we'll get back to you."
  fields={fields}
  schema={userSchema}
  onSubmit={handleSubmit}
  trigger={<Button>Open Form</Button>}
/>
```

## Props Reference

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `title` | `string` | - | **Required.** Dialog title |
| `fields` | `FieldConfig[]` | - | **Required.** Array of field configurations |
| `schema` | `z.ZodObject<any>` | - | **Required.** Zod validation schema |
| `onSubmit` | `(data: any) => Promise<void> \| void` | - | **Required.** Form submission handler |
| `description` | `string` | - | Dialog description text |
| `trigger` | `React.ReactNode` | - | Trigger element for dialog |
| `open` | `boolean` | - | Controlled dialog open state |
| `onOpenChange` | `(open: boolean) => void` | - | Controlled dialog open state handler |
| `submitButtonText` | `string` | `"Submit"` | Submit button text |
| `cancelButtonText` | `string` | `"Cancel"` | Cancel button text |

## Field Configuration

Each field in the `fields` array should have the following structure:

```tsx
interface FieldConfig {
  name: string;              // Field name (must match schema key)
  label: string;             // Field label
  type: "text" | "email" | "tel" | "textarea" | "file";
  placeholder?: string;      // Placeholder text
  validation: z.ZodType<any>; // Zod validation schema
}
```

## Examples

### Basic Contact Form

```tsx
import { DynamicDialogForm } from "@/components/organisms/dialog-form/dialog-form";
import { z } from "zod";
import { Button } from "@/components/ui/button";

const contactSchema = z.object({
  name: z.string().min(2, "Name too short"),
  email: z.string().email("Invalid email"),
  message: z.string().min(10, "Message too short"),
});

const contactFields = [
  {
    name: "name",
    label: "Name",
    type: "text" as const,
    placeholder: "Your name",
    validation: contactSchema.shape.name,
  },
  {
    name: "email",
    label: "Email",
    type: "email" as const,
    placeholder: "you@example.com",
    validation: contactSchema.shape.email,
  },
  {
    name: "message",
    label: "Message",
    type: "textarea" as const,
    placeholder: "Your message...",
    validation: contactSchema.shape.message,
  },
];

export function ContactForm() {
  const handleSubmit = async (data: any) => {
    await fetch("/api/contact", {
      method: "POST",
      body: JSON.stringify(data),
    });
  };

  return (
    <DynamicDialogForm
      title="Contact Us"
      description="Send us a message"
      fields={contactFields}
      schema={contactSchema}
      onSubmit={handleSubmit}
      trigger={<Button>Contact Us</Button>}
    />
  );
}
```

### Form with File Upload

```tsx
import { DynamicDialogForm } from "@/components/organisms/dialog-form/dialog-form";
import { z } from "zod";

const uploadSchema = z.object({
  title: z.string().min(3, "Title too short"),
  file: z.instanceof(File).refine(
    (file) => file.size <= 5 * 1024 * 1024,
    "File must be under 5MB"
  ),
});

const uploadFields = [
  {
    name: "title",
    label: "Document Title",
    type: "text" as const,
    placeholder: "Enter document title",
    validation: uploadSchema.shape.title,
  },
  {
    name: "file",
    label: "Upload File",
    type: "file" as const,
    validation: uploadSchema.shape.file,
  },
];

export function UploadForm() {
  const handleUpload = async (data: any) => {
    const formData = new FormData();
    formData.append("title", data.title);
    formData.append("file", data.file);
    
    await fetch("/api/upload", {
      method: "POST",
      body: formData,
    });
  };

  return (
    <DynamicDialogForm
      title="Upload Document"
      description="Upload a document (max 5MB)"
      fields={uploadFields}
      schema={uploadSchema}
      onSubmit={handleUpload}
      trigger={<Button>Upload</Button>}
    />
  );
}
```

### Complex Validation

```tsx
import { z } from "zod";

const complexSchema = z.object({
  username: z.string()
    .min(3, "Username must be at least 3 characters")
    .max(20, "Username must be at most 20 characters")
    .regex(/^[a-zA-Z0-9_]+$/, "Only letters, numbers, and underscores"),
  
  email: z.string()
    .email("Invalid email")
    .refine(
      (email) => email.endsWith("@company.com"),
      "Must use company email"
    ),
  
  password: z.string()
    .min(8, "Password must be at least 8 characters")
    .regex(/[A-Z]/, "Must contain uppercase letter")
    .regex(/[0-9]/, "Must contain number"),
  
  confirmPassword: z.string(),
}).refine(
  (data) => data.password === data.confirmPassword,
  {
    message: "Passwords don't match",
    path: ["confirmPassword"],
  }
);

const complexFields = [
  {
    name: "username",
    label: "Username",
    type: "text" as const,
    validation: complexSchema.shape.username,
  },
  {
    name: "email",
    label: "Email",
    type: "email" as const,
    validation: complexSchema.shape.email,
  },
  {
    name: "password",
    label: "Password",
    type: "text" as const,
    validation: complexSchema.shape.password,
  },
  {
    name: "confirmPassword",
    label: "Confirm Password",
    type: "text" as const,
    validation: complexSchema.shape.confirmPassword,
  },
];
```

## Validation Patterns

### String Validation

```tsx
z.string()
  .min(3, "Minimum 3 characters")
  .max(50, "Maximum 50 characters")
  .email("Invalid email")
  .url("Invalid URL")
```

### Number Validation

```tsx
z.number()
  .min(0, "Must be positive")
  .max(100, "Must be less than 100")
  .int("Must be an integer")
```

### File Validation

```tsx
z.instanceof(File)
  .refine(
    (file) => file.size <= 5 * 1024 * 1024,
    "File must be under 5MB"
  )
  .refine(
    (file) => ["image/jpeg", "image/png"].includes(file.type),
    "Only JPEG and PNG files allowed"
  )
```

### Optional Fields

```tsx
z.string().optional()
z.string().nullable()
z.string().optional().nullable()
```

## Best Practices

1. **Always provide meaningful error messages** in your Zod schemas
2. **Use appropriate field types** for better UX (email for emails, tel for phones)
3. **Handle file uploads properly** with size and type validation
4. **Use controlled mode** when you need external control over dialog state
5. **Reset form state** after successful submission (built-in)
6. **Provide loading feedback** for async operations (built-in)
7. **Validate on the server** even with client-side validation

## Troubleshooting

### Common Issues

**Form not validating**
- Ensure field names match schema keys exactly
- Check that validation schemas are properly defined
- Verify Zod schema structure matches your data

**File upload not working**
- Ensure FileUpload component is properly imported
- Check file size and type validation rules
- Verify FormData handling in submit handler

**Dialog not opening**
- Check if trigger prop is provided for uncontrolled mode
- For controlled mode, ensure open and onOpenChange are provided
- Verify Dialog component is properly styled

**Validation errors not showing**
- Ensure validation returns error messages
- Check that field names are unique
- Verify Zod schema validation logic

## Performance Considerations

### Large Forms

For forms with many fields (>10), consider:
- Lazy loading field components
- Debouncing validation for text inputs
- Using memo for field configurations
- Optimizing re-renders with proper dependencies

### File Uploads

For large file uploads:
- Implement progress indicators
- Use chunked upload for very large files
- Add file type validation on client and server
- Consider compression for images
