# DataTable Component

Comprehensive, reusable data table component built on top of [TanStack Table](https://tanstack.com/table) and [shadcn/ui](https://ui.shadcn.com/). Provides enterprise-grade features for displaying and managing tabular data with minimal setup.

## Features

### Core Functionality
- **Sorting** - Click column headers to sort data ascending/descending
- **Filtering** - Global search across multiple columns with debounced input
- **Pagination** - Client-side and server-side pagination support
- **Row Selection** - Single or multiple row selection with checkboxes
- **Column Visibility** - Dynamic show/hide columns via dropdown
- **Row Actions** - Custom dropdown actions per row (edit, delete, view, etc.)

### Visual Enhancements
- **Striped Rows** - Alternating row colors for better readability
- **Hover Effects** - Row hover states for better UX
- **Sticky Headers** - Keep headers visible while scrolling
- **Loading States** - Built-in skeleton loader
- **Empty States** - Customizable empty state messages
- **Responsive Design** - Mobile-friendly table layout

### Advanced Features
- **Zustand Integration** - State management for multiple table instances
- **Toolbar Actions** - Custom toolbar components at various positions
- **Server-Side Support** - Ready for server-side pagination/filtering/sorting
- **TypeScript** - Full type safety with generics
- **Custom Cell Renderers** - Flexible cell content rendering

## Quick Start

```tsx
import { DataTable, createColumn, createSortableColumn } from "@/components/organisms/@tanstack-react-table/data-table";

interface User {
  id: string;
  name: string;
  email: string;
  role: string;
}

const columns = [
  createColumn<User>("id", "ID", "id"),
  createSortableColumn<User>("name", "Name", "name"),
  createColumn<User>("email", "Email", "email"),
  createColumn<User>("role", "Role", "role"),
];

const data: User[] = [
  { id: "1", name: "John Doe", email: "john@example.com", role: "Admin" },
  { id: "2", name: "Jane Smith", email: "jane@example.com", role: "User" },
];

export default function UserTable() {
  return (
    <DataTable
      tableName="users"
      columns={columns}
      data={data}
      searchKey={["name", "email"]}
      searchPlaceholder="Search users..."
    />
  );
}
```

```tsx
  category: string;

  stock: number;
  status: "active" | "inactive" | "discontinued";
  createdAt: string;
}

  createColumn<Product>("id", "ID", "id", ({ getValue }) => (
    <div className="font-mono text-sm">{getValue() as string}</div>
  )),
  
  createSortableColumn<Product>("name", "Product Name", "name", ({ getValue }) => (
    <div className="font-medium">{getValue() as string}</div>
  )),
  
  createColumn<Product>("category", "Category", "category", ({ getValue }) => (
    <Badge variant="secondary">{getValue() as string}</Badge>
  )),
  
  createSortableColumn<Product>("price", "Price", "price", ({ getValue }) => {
    const price = getValue() as number;
    return (
      <div className="font-medium">
        Rp {price.toLocaleString("id-ID")}
      </div>
    );
  }),
  
  createSortableColumn<Product>("status", "Status", "status", ({ getValue }) => {
    const status = getValue() as string;
    return (
      <Badge

          status === "active"
            ? "default"
            : status === "inactive"
              ? "secondary"
              : "destructive"
        }
      >
        {status}
      </Badge>
    );
  }),
];

export default function ProductTable() {
  const handleRowAction = (action: string, row: any) => {
    switch (action) {
      case "view":
        console.log("View:", row.original);
        break;
      case "edit":
        console.log("Edit:", row.original);
        break;
      case "delete":
        console.log("Delete:", row.original);
        break;
    }
  };

  return (
    <DataTable
      tableName="products"
      columns={columns}
      data={products}
      searchKey={["name", "category"]}
      searchPlaceholder="Search products..."
      enableRowSelection={true}
      enableColumnVisibility={true}
      enableSorting={true}
      enableSearch={true}
      enablePagination={true}
      pageSize={10}
      onRowAction={handleRowAction}
      rowActions={[
        {
          label: "View Details",
          value: "view",
          icon: <Eye className="h-4 w-4" />,
        },
        {
          label: "Edit Product",
          value: "edit",
          icon: <Edit className="h-4 w-4" />,
        },
        {
          label: "Delete Product",
          value: "delete",
          icon: <Trash2 className="h-4 w-4" />,
          variant: "destructive",
        },
      ]}
      emptyMessage="No products found."
      isStriped={true}
      isHoverable={true}
      isStickyHeader={true}
    />
  );
}
```

## State Management with Zustand

The component uses Zustand for state management, allowing multiple table instances without conflicts:

```tsx
import { useReactTableData } from "@/hooks/use-react-table";

function MyComponent() {
  const {
    originalData,
    filteredData,
    selectedRows,
    setFilteredData,
    resetTable,
  } = useReactTableData("myTable");

  // Filter data programmatically
  const filterByStatus = (status: string) => {
    setFilteredData(
      originalData.filter(item => item.status === status)
    );
  };

  // Reset to original data
  const handleReset = () => {
    resetTable();
  };

  return (
    <>
      <DataTable
        tableName="myTable"
        columns={columns}
        data={data}
      />
      <Button onClick={() => filterByStatus("active")}>
        Show Active Only
      </Button>
      <Button onClick={handleReset}>
        Reset
      </Button>
    </>
  );
}
```

## Server-Side Pagination

For large datasets, use server-side pagination:

```tsx
const [queryOptions, setQueryOptions] = useState<DataTableQuery>({
  page: 1,
  limit: 10,
  offset: 0,
  totalPage: 0,
  lastPage: 0,
  search: "",
});

const fetchData = async () => {
  const response = await fetch(`/api/data?page=${queryOptions.page}&limit=${queryOptions.limit}&search=${queryOptions.search}`);
  const result = await response.json();
  setData(result.data);
  setQueryOptions(prev => ({
    ...prev,
    totalPage: result.pagination.totalPages,
    lastPage: result.pagination.totalPages,
  }));
};

useEffect(() => {
  fetchData();
}, [queryOptions.page, queryOptions.limit, queryOptions.search]);

return (
  <DataTable
    tableName="server-table"
    columns={columns}
    data={data}
    queryOptions={queryOptions}
    onPageChange={(page) => setQueryOptions(prev => ({ ...prev, page }))}
    onSearchChange={(search) => setQueryOptions(prev => ({ ...prev, search, page: 1 }))}
  />
);
```

## Performance Considerations

### Large Datasets

For large datasets (>1000 rows), consider:
- Using server-side pagination
- Implementing virtual scrolling
- Debouncing search inputs (built-in with 500ms debounce)
- Memoizing column definitions

## Troubleshooting

### Common Issues

**Columns not rendering**
- Check that `accessorKey` matches your data properties
- Ensure data is not empty or undefined

**Sorting not working**
- Verify `enableSorting` is true
- Use `createSortableColumn` for sortable columns
- Check that `accessorKey` is properly set

**Search not working**
- Ensure `searchKey` matches a column's `accessorKey`
- Check that `enableSearch` is true
- Verify search key is an array if searching multiple fields

**Row actions not showing**
- Verify `rowActions` array is not empty
- Check that `onRowAction` callback is provided

**Multiple tables conflicting**
- Ensure each table has a unique `tableName`
- Different table names prevent Zustand store conflicts

## Best Practices

1. **Always provide a unique `tableName`** for each table instance
2. **Use `createSortableColumn`** for columns that need sorting
3. **Memoize column definitions** to prevent unnecessary re-renders
4. **Use server-side pagination** for datasets > 1000 rows
5. **Provide meaningful `emptyMessage`** for better UX
6. **Use `toolbarActions`** for custom toolbar functionality
7. **Implement proper loading states** for async data
